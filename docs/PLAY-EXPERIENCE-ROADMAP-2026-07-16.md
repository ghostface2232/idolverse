# 플레이 경험 구현 로드맵 — "편집" 4마일스톤

작성일: 2026-07-16
근거 문서: `docs/idolverse_final_analysis.md` (플레이 경험 최종 분석, 7장 우선순위)
관련 문서: `docs/MAIN-GAME-UX-ARCHITECTURE-PLAN-2026-07-14.md`, `docs/DESIGN-living-diorama.md`

## 0. 이 문서의 위치와 정합 원칙

이 로드맵은 분석 문서의 최우선 5개 항목을 **플레이어 체감 단위의 마일스톤 4개**로 재구성한 콘텐츠 편집 트랙이다. 기존 문서와 다음처럼 맞물린다.

- `MAIN-GAME-UX-ARCHITECTURE-PLAN`의 Phase 1(WeekDelta·주간 상태 머신)과 Phase 2(GoalStrip 등 GameShell)는 이 로드맵이 **소비하는 인프라**다. 이 로드맵 전체가 사실상 그 문서의 "핵심 콘텐츠 연결" 단계의 상세 설계다.
- `DESIGN-living-diorama.md`의 원칙 **"연출은 콘텐츠를 앞지르지 않는다"**를 따른다. 이 로드맵의 모든 연출(개봉 시퀀스 등)은 presentationBus 명령 + React 오버레이까지만 만들고, Phaser 스프라이트 고도화는 디오라마 로드맵의 몫으로 남긴다. 모든 사건은 WeekDelta(source.kind + day)에 기록되므로 나중에 디오라마가 같은 데이터를 소비하면 된다.
- **결정론 불변 규칙 유지**: 새로 추가되는 사건 감독(pacing director)·프로젝트 전이·이정표 판정은 전부 processWeek 내부에서 시드 기반으로 계산한다.

### 확정된 설계 결정: 전속계약 2~3년

K-POP 실제 관습은 7년이지만 게임 페이싱에는 너무 길다. **기본 3년(156주, balance 상수로 조정 가능)**으로 설정한다.

- 계약 만료가 캠페인의 시계가 된다: 1년차 데뷔~신인상 → 2년차 성장 → 2~3년차 말 첫 재계약 협상 웨이브.
- 멤버별 인기도·만족도·개인 브랜드에 따른 **재계약 협상**이 FM식 인력 운영 컨텐츠의 중심축이 된다(후반 티어).
- M1에서는 장기 목표 레인에 "계약 만료 D-day" 시계만 심는다. 협상 로직은 만들지 않는다.

## 1. 마일스톤 분할 — 순서와 근거

기술 의존 순서: **이정표/phase 전이(공통 기반) → 단계형 프로젝트 프레임(데뷔에서 최초 구축) → 프레임 재사용(컴백)**. 기회 카드는 독립적이고 작아서 데뷔 프로젝트 직후에 넣으면 초반·중반 양쪽의 "조용한 주" 문제를 한 번에 해소한다.

| 순서 | 마일스톤 | 완료 후 플레이어가 새로 느끼는 것 | 분석 문서 7장 매핑 |
|---|---|---|---|
| M1 | **나침반** — 3시간축 목표·해금 체인·훈련 투명화 | "이번 주에 뭘 왜 하는지 안다. 훈련 선택이 처음으로 고민된다" | 최우선 4, 5 |
| M2 | **첫 드라마** — 데뷔 20주 단계형 프로젝트 + 사건 감독 + 차트 개봉 | "데뷔를 향해 전진하고 있고, 첫 세션 안에 성취의 순간들이 온다" | 최우선 1 (+기반 정리) |
| M3 | **유혹** — 기회 카드 레인 | "평온한 주에도 '받을까 말까'가 있다. 매주의 질감이 달라졌다" | 최우선 3 |
| M4 | **심장** — 컴백 프로젝트 상위 루프 + 중첩 대기 | "발매 긴장→개봉 쾌감→정산→다음 기획 — 세션을 끊기 어렵다" | 최우선 2 |

## 2. M1 「나침반」 — 목표 체계와 훈련 투명화

**정의**: 플레이어가 화면을 보고 5초 안에 "이번 주 할 일 / 지금 프로젝트의 진행률 / 다음에 열릴 것"을 읽고, 훈련 옵션마다 효용과 대가를 누르기 전에 안다.

### 구현 스케치

신규:
- `src/systems/progressionSystem.ts` — 이정표 평가와 phase 전이 판단의 단일 소스. weekProcessor 주진행 직후 새 번호 단계로 삽입, 달성·전이를 WeekDelta(`source.kind: 'milestone'`)로 기록.
- `src/data/milestones.ts` — 잠자는 상수의 공개 이정표화. `AWARD_ELIGIBILITY_THRESHOLDS`(본상/대상), `GLOBAL_EXPANSION_REQUIREMENTS`(글로벌 35/50/75), 프로모션 requirements를 참조하는 이정표 정의 + "달성 → 해금되는 행동" 체인(분석 문서 4.3의 사다리).
- `src/stores/progressionStore.ts` — 달성 기록·활성 목표 3레인. 스냅샷 브리지 5곳(GameSnapshot/buildGameSnapshot/applyGameSnapshot/toPersistedSnapshot/resetAllStores) 갱신.

확장:
- `GoalStrip.tsx` — 3 prop 단일 표시 → 3레인(이번 주/현재 프로젝트/장기) 모델 표시. 각 목표에 현재치·목표치·남은 시간·해금 보상.
- `GameDashboard.tsx` — 투자자 조건 하드코딩 조립을 progression 셀렉터로 교체. 투자자 조건은 장기 레인의 항목으로 흡수.
- `balance.ts` — `CONTRACT_TERM_WEEKS`(기본 156) 신설, 장기 레인에 계약 D-day 노출.
- `trainingSystem.ts` — 지배 선택 해소:
  - 휴식일: 무비용 스트레스 감소 → 성장량 감소 트레이드오프 도입.
  - 개인레슨: 성장 0 → 포커스 스탯 집중 성장 + 팀 케미 기회비용.
  - 예능: 확정 대중성 → 시드 기반 분산(바이럴/무반응).
  - 성장식이 순수 함수 곱이므로 **예측 함수를 추출**해(같은 상수·같은 함수 재사용) 훈련 UI에 "예상 성장 / 스트레스 / 부상 확률 / 포기하는 것" 프리뷰 표시.

의도적으로 안 하는 것: 육성 8종 준비상태 전면 재편(멤버별 스케줄 맵 구조 변경 필요), mental 훈련 경로(M2 미디어 준비 스테이지에서), potential 소프트캡.

### 플레이 검증 기준
- 신규 세이브 로드 후 5초 안에 3레인 목표가 읽힌다.
- 모든 훈련 옵션에 예상 결과가 사전 표시되고 실제 주간 결과와 방향이 일치한다.
- "휴식일 항상 켜기 / 예능 반복"이 상황 무관 최적해가 아니다.
- balance.ts에만 있던 시상식·글로벌 요건이 장기 레인에 이정표로 보인다.

## 3. M2 「첫 드라마」 — 데뷔 20주 프로젝트 + 사건 감독 + 개봉 연출

**정의**: 첫 세션이 "주차 소비"가 아니라 6개의 미니 시즌으로 느껴지고, 분석 문서 3.2의 첫 순간들(10분 강점 발견 → 45분 쇼케이스)이 시드와 무관하게 보장된다.

### 구현 스케치

신규:
- `src/systems/projectSystem.ts` — **범용 단계형 프로젝트 프레임**(M4에서 컴백이 재사용하므로 처음부터 범용으로: stages[], 스테이지별 주차 창, 진입 게이트, 진입 시 스폰할 이벤트/성취 공개, 복수 인스턴스 허용).
- `src/data/debutProject.ts` — 분석 문서 3.2 표의 6단계(1-3주 관찰 → 4-6주 포지션 평가전·곡 후보 → 7-9주 컨셉 테스트 → 10-13주 타이틀 결정 → 14-17주 쇼케이스 리허설 → 18-20주 데뷔 프로모션).
- `src/systems/pacingDirector.ts` — 확률 이벤트 위의 최소 발생 보장 레이어("첫 개인 사건 2-4주, 첫 라이벌 등장 8-12주, 첫 공개 평가 12-16주"를 시드 기반 강제 스폰). 기존 eventSystem·RANDOM_EVENT_POOL 조건 체계를 소비.

확장:
- `data/events.ts` — 스테이지 이벤트 추가(포지션 평가전, 컨셉 테스트 반응, 유닛 연습 케미 발견, 쇼케이스 리허설 사고 등).
- `balance.ts` — `DEBUT_REQUIREMENTS`(준비도 80·평균보컬 55·최소 14주) 신설 → 이정표로 노출, GoalStrip 프로젝트 레인에 게이지 상시 표시.
- `PositionAssignment.tsx` — 확정 배정 → **가배정**, 평가전 이벤트가 적합도 공개 + 1회 재조정 기회(3.1절 이연의 최소 슬라이스).
- phase 전이 training→debut를 쇼케이스 결과 게이트로 연결 — 기존 phase 조건 코드들이 자동 개통됨.

기반 정리 (M2 초입에 수행):
- `evaluationSystem.evaluateRelease` → weekProcessor 발매 단계 배선. 데뷔 앨범이 첫 소비자.
- `albumStore.releaseAlbum`의 자체 간이 품질 공식 제거 → `albumSystem.calculateAlbumQuality` 단일 소스화. 차트 카운트다운 연출이 chartRank를 신뢰해야 하는 시점이기 때문.
- `weeklyChartDecay` 주간 배선.

연출: presentationBus로 `chart-reveal` 명령 → React 오버레이(아래에서부터 훑는 카운트다운 시트). weekRunner의 event_focus 상태 재사용. Phaser 작업 없음.

### 플레이 검증 기준 (분석 문서 3.2를 완료 조건으로 채택)
- 첫 10분: 멤버 강점 발견 / 첫 20분: 개인 사건 / 첫 30분: 공개 평가 / 첫 45분: 쇼케이스 결과 / 첫 세션 내: 데뷔 + 차트 진입 카운트다운.
- GoalStrip 프로젝트 레인이 항상 "6단계 중 어디"와 데뷔 조건 게이지를 표시.
- 시드 고정 시 20주 리플레이가 결정론적으로 동일(사건 감독 포함).

## 4. M3 「유혹」 — 기회 카드 레인

**정의**: 위기 0건인 건강한 주에도 "받을까 말까"의 능동 결정이 존재하고, 거절이 유효한 전략인 비대칭(위기=의무, 기회=선택)이 생긴다.

### 구현 스케치
- `types/game.ts` — `WeeklyDecisionTriggerKind`에 기회 kind, WeeklyDecision에 `lane: 'crisis' | 'opportunity'`와 만료(미선택 시 소멸) 필드.
- `generateWeeklyDecisionCards.ts` — 기존 build*Card 팩토리 패턴으로 buildOpportunity*Card 추가. 빈도 제어(위기 카드 수·최근 기회 빈도·phase 조건으로 억제).
- `src/data/opportunities.ts`(신규) — 예능 섭외, 광고 제안, 콜라보, 커버 영상 바이럴 + **라이벌 컴백 겹침 감지 카드 1종**(정면 승부/연기/차별화 — 라이벌 대응의 맛보기).
- 카드 UI — 레인 시각 구분, 효과에 기회비용 명시(예능 수락 시 그 주 훈련 스킵). M1의 예측 표시 문법 재사용.

### 플레이 검증 기준
- 위기 0건인 주 기준 2~3주에 1회꼴로 기회 카드가 등장하고 무시해도 페널티가 없다.
- 기회 카드를 거절하는 것이 명백히 옳은 상황(피로 누적, 컴백 직전)이 실제 발생한다.
- 데뷔 준비기와 성장기에서 phase 조건에 맞는 다른 기회 풀이 나온다.

## 5. M4 「심장」 — 컴백 프로젝트 상위 루프

**정의**: 중반 플레이가 병렬 메뉴 소비가 아니라 "지금 컴백 사이클의 어느 단계"라는 맥락 속 결정이 되고, 발매 개봉 → 정산 → 다음 기획 착수의 중첩 대기로 세션 종료 지점이 사라진다.

### 구현 스케치
- `src/data/comebackProject.ts`(신규) — M2의 projectSystem 프레임에 얹는 컴백 스테이지: 컨셉 조사 → 곡 후보(generateTitleTrackCandidates 재사용) → 타이틀 결정 → 파트 분배 → 연습 → 티저 → 사전 반응 → 발매 → 음악방송 → 정산·회고. 새 시스템 파일이 거의 필요 없는 것이 M2에서 범용 프레임을 만든 보상.
- 중간 공개: `data/events.ts`에 스테이지 이벤트(녹음 사고, 안무 시안 선택, 티저 반응). 사전 반응 스테이지에서 `calculateFandomExpectation`을 **플레이어에게 미리 보여주는** 용도로 노출 — 이미 있는 계산을 정보 콘텐츠로 전환.
- 개봉 연출 재사용: M2의 chart-reveal + 음악방송 1위 후보 대결(evaluationSystem 차트풀 + competitorSystem 컴백 시뮬이 상대를 공급).
- 중첩 대기: 활동기 후반에 다음 앨범 "컨셉 조사" 스테이지 개방(projectSystem 복수 인스턴스). GoalStrip 프로젝트 레인이 D-day 카운트로 전환.
- 정산·회고: WeekReport 격상에 컴백 정산 뷰 — 투자사 목표 대비, 팬덤 4축 변화, 다음 사이클 훅.
- phase 전이 완성: debut→growth(첫 컴백 정산), growth→peak(수상·글로벌 이정표 게이트).

### 플레이 검증 기준
- 발매 4주 전부터 매주 결정에 "컴백 D-n" 맥락이 표시된다.
- 발매 주에 개봉 시퀀스 후 정산 화면이 다음 훅을 제시한다.
- 정산 직후 다음 앨범 기획을 바로 시작할 수 있다(끊기 좋은 지점의 소멸).
- 두 번째 컴백에서 팬 기대치(컨셉 히스토리)가 첫 번째와 다른 질문을 만든다.

## 8. P1 「기초 체력」 밸런스 개편 — 2026-07-17 반영

M4 완료 후 시뮬레이션 프로브(음악방송 승률 전 구간 0%, 성장 축 스케일 버그, 유령 스태프, 무비용 컴백)에서 출발해, 캠페인 전체의 성장 아크를 확정한 개편. 아래 결정은 이후 마일스톤의 전제다.

### 확정된 설계 결정

- **성장 아크**: 1~2년차 = 같은 신인 코호트 경쟁 → 신인상(데뷔 후 2년차 말까지, 이후 자격 소멸). 3년차부터 전체 그룹 경쟁으로 전환하고, 기성 그룹 역전(차트 1위·음악방송 1위 상시화)은 **플레이 능숙도에 따라 3~7년차**. 디스코그래피 보너스 같은 자동 누적 우위는 두지 않는다.
- **출시는 막지 않는다 (Game Dev Story 원칙)**: 데뷔·컴백 모두 완성도가 낮다고 발매가 막히지 않는다. 낮은 준비도는 품질 progressMult·쇼케이스 등급·차트 결과로 귀결된다. 데뷔는 창단 시 일정 티어(속전속결 16주/정석 20주/완성형 24주)로 정하며, 빠르면 첫 신인 프리미엄(주목도 배율), 늦으면 완성도 대신 경쟁 신인 데뷔에 가려질 확률을 감수한다.
- **FM 유스식 육성**: 신인 능력치 상한 30~40대(`RECRUIT_STAT_BANDS`), 예산·스카우트는 능력치가 아니라 잠재력을 산다(`RECRUIT_POTENTIAL`). 잠재력이 성장 상한을 정하고(`POTENTIAL_STAT_CAP`: 0.8→67 ~ 1.7→99) 상한 근처에서 taper. 주간 성장은 포커스 스탯 기준 연 +20~25로, 고잠재 멤버도 캡 도달은 4~5년차.
- **스태프도 같은 원리**: 창단 풀 능력 상한 60, 실무 성장 연 +3 수준, 채용 시 랜덤 잠재 상한(`STAFF_GROWTH`) — 상한에 닿으면 더 크지 않으므로 결국 재모집(M5)이 필요하다. 유령 기본값(부재 시 능력 30) 폐지: 프로듀서 부재는 후보곡 품질 붕괴 + 주간 제작 사고(`PRODUCTION_RISK`)로 나타난다.
- **경쟁 산식 정합**: 플레이어 chartPower에 팬덤·글로벌 축이 실스케일로 반영(성장으로 이기는 구조의 수학적 성립), 배경 그룹 파워 ×0.8, 음악방송은 음원 70% + 팬덤 투표 30%(`MUSIC_SHOW_SCORE`).
- **컴백 제작비**: 예산 3단(절약 3천만/표준 6천만/대형 1.2억)이 사전 제작분을 결정 — "정산 즉시 무조건 재컴백" 지배 선택 제거.

### 페이싱 원칙 (전 마일스톤 공통)

1. 스펙은 한 번에 오르지 않는다 — 느리지만 매주 눈에 보이는 성장과 촘촘한 중간 보상(이정표·스테이지 진입·신인상)을 준다.
2. 잠재력은 보장이 아니다 — 낮은 천장, 과로·스트레스, 콘셉트 부적합으로 발휘되지 못할 수 있다.
3. 이슈는 사전 신호(스트레스·만족도·조건 격차·능력 미달 경고) 후에 발생하고, 절망적이지도 무시 가능하지도 않은 난이도로 준다.

### 측정 결과 (시드 프로브)

- 성장: 잠재력 0.9 → Y1 40, Y4 70(캡) / 1.2 → Y1 46, Y3 80 / 1.6 → Y1 54, Y3 95.
- 음악방송 승률: 1년차 0% → 2년차 5% → 3-4년차(상위 플레이 프로필) 82% → 5년차+ 98%.
- 데뷔 일정: 속전속결 품질 79·쇼케이스 62 ↔ 완성형 품질 89·쇼케이스 70. 무프로듀서 품질 76·쇼케이스 57.

### P2 「활동기」 패치 — 같은 날 반영 (REVIEW-2026-07-17 처방)

- **활동기 신설**: 컴백 사이클 14주 → 16주. 발매 후 3주 활동기 스테이지에서 음악방송이 매주 열리고(후보권 진입 시 1위 대결, 1위 횟수 누적), **프로모션 실행이 처음으로 배선**됐다(주당 1건, 활동기 전용) — 이정표 해금 체인(팬사인회~돔)이 실제 콘텐츠가 됐다.
- **음악방송 후보권**: 차트 15위 이내만 1위 후보에 선다. 밖이면 조용한 무대 리포트, "첫 1위 후보 진입"이 이정표(`musicShowCandidacies`)로 추가 — 지는 대결 연출의 반복 제거.
- **신인상 정상화**: 라이벌 시상 지표를 연간 최고 품질(`seasonBestQuality`) 기반으로 교체. 측정: 자동 수상(2년차 미디언 100%) → 1년차 0% / 2년차 미디언 3~30% / 2년차 상위 100% — 신인 코호트 경쟁이 실제 경쟁이 됐다.
- **시장 스윙**: 앨범 성적이 실력의 선형 함수가 되지 않도록, 죽어 있던 트렌드 파라미터를 배선하고 계절 적합·발매 운을 차트 파워에 곱한다(`RELEASE_MARKET_SWING`). 측정: 같은 실력에서 트렌드 적중+여름 청량 ×1.22~1.49 ↔ 약세 정면충돌 ×0.76~0.93 — 최대 2배 격차. 성장은 기대값을 올릴 뿐 결과를 보장하지 않는다.
- **데뷔 지연 상대화**: 만족도 페널티가 고정 20주가 아니라 창단 시 약속한 일정 대비 지연으로 판정.
- **캠페인 시드**: 주차 기반 세계 RNG(경쟁 시뮬·이벤트 라이벌 스폰)에 회차 시드를 섞어 회차마다 세계가 다르게 진화한다. 구버전 세이브(시드 0)는 기존과 동일.

## 9. M5 「사람」 — 재계약·인사·시설 (다음 마일스톤 설계)

**정의**: 중후반 플레이의 축이 "무엇을 발매하나"에서 "누구와 함께 가나"로 확장된다. P1의 느린 성장 곡선과 잠재 상한이 만든 개인 격차가 계약 조건·불만·세대교체라는 FM식 인력 운영 컨텐츠로 전환된다.

### 구현 스케치

- **개인 인기도 축 (선행 조건)**: 멤버별 인기도를 신설(직캠 바이럴, 예능·OST 개인 활동, 포지션 노출이 축적). 재계약 조건과 팀 내 격차 서사의 근거 데이터.
- **멤버 재계약**: 성장·인기 급상승 시 주기(52~104주)가 앞당겨지는 재계약 요구. 조건 협상(수익 배분·개인 활동 보장)은 결정 카드/모달로. **팀 내 조건 격차가 커지면 낮은 조건의 멤버가 불만을 품고 재계약을 요구**(satisfactionSystem에 격차 페널티 확장). 결렬 시 이탈·이적 — `calculateMemberLeaveImpact` 재사용.
- **스태프 재계약·재모집**: 자본·업계 신뢰 게이트로 상위 풀 해금(`generateStaffCandidates`의 `maxAbility` 인상 + `staffRefresh` 상시화). 잠재 상한에 닿은 스태프의 교체 결정 — 오래 함께한 스태프 해고에는 팀 만족도 대가를 붙인다.
- **시설 상시 업그레이드 노출**: `financeStore.upgrade` 액션은 이미 존재 — 창단 이후 대시보드에서 접근 가능한 UI가 핵심(Game Dev Story의 사무실 업그레이드처럼 성장의 가시적 보상).
- 전속계약 만료(156주) 시계는 M1부터 장기 레인에 있음 — 팀 단위 재계약 웨이브가 이 컨텐츠의 최종 시험대.

### 플레이 검증 기준

- 2년차 이후 "가장 인기 있는 멤버"와 "가장 조건이 낮은 멤버"가 데이터로 구분되고, 그 격차가 실제 사건(재계약 요구·불만)을 만든다.
- 잠재 상한에 닿은 스태프를 교체할지, 함께 갈지가 자금·만족도 트레이드오프가 있는 실제 결정이 된다.
- 시설 업그레이드가 창단 이후에도 접근 가능하고, 업그레이드 직후 주간 리포트에서 효과가 읽힌다.

### 구현 반영 (2026-07-17)

- **개인 인기도**(`popularitySystem`): 활동기 노출(포지션 가중)·개인 예능·프로모션 참가·음방 1위·직캠 바이럴로 쌓이고 비활동 주에 감쇠. 멤버 현황에 인기 수치와 "인기 1위" 배지 표시.
- **성격 태그**(`TEMPERAMENT_PROFILES`): 야심가/헌신형/안정형/섬세형 — 격차 민감도·재계약 주기 보정·과로 민감도를 가른다. 채용 카드와 멤버 현황에 표시.
- **멤버 재계약**: 주기(78주±성격 보정) 도래 또는 조기 트리거(인기가 처우를 앞지름·과로)로 위기 카드 등장. 조건 인상(계약금 = 기본 + 인기×150만)/보너스/동결 3지선다 — 동결은 성격 배율이 붙은 만족도 대가와 조기 재협상으로 돌아온다. 처우 등급 갱신은 weekProcessor가 카드 id로 처리(emergency-investor 패턴).
- **조건 격차 불만**: 팀 내 최고 처우와 2등급 이상 벌어지면 주간 만족도 페널티(야심가 ×2, 헌신형 ×0.5).
- **이탈 구현**: 만족도 바닥(이탈 임박)이 4주 연속이면 실제 탈퇴 — `calculateMemberLeaveImpact`로 팬덤·업계 타격. 최소 3인은 지킨다(팀 해체 방지).
- **스태프 상시 시장**: 더보기 → 인사 관리. 풀 상한 = 60 + 업계 신뢰×0.5, 후보 매주 회전, 성장 여력은 범위 라벨(성장 정체/성장 여지/대기만성)로만 노출 — 스카우트의 안개. 교체는 팀 만족도 -4.
- **시설 상시 투자**: 더보기 → 시설 투자. 숙소/연습실/장비 레벨업 + 메디컬/보안 도입이 창단 이후에도 열린다.
- 미룬 것: 개인 활동 보장형 협상 옵션(개인 스케줄 시스템과 함께), 타사 이적 서사(단순 탈퇴로 대체), 연습생 재모집(스태프만 상시화).

### 멤버 특성·앨범별 센터 — 같은 날 추가 반영

- **인간적 특성**(`data/memberTraits.ts`): 멤버에게 장르 태그(Y2K·섹시) 대신 성격(순수·귀여움·도도·활력·차분)과 인상(고양이상·강아지상·장신·세련·신비·건강미)이 붙는다. 컨셉 친화(conceptAffinity)는 특성에서 파생되는 캐시 — 기존 시스템(품질·훈련 보너스·만족도)은 그대로 소비한다. 구버전 세이브는 id 해시 시드로 특성을 받고 친화를 재파생한다.
- **조합 시너지**: 성격과 인상이 모두 컨셉을 지지하면(도도×고양이상×섹시) 센터 조합 보너스 +12 — "뜰 확률이 높은 조합"의 근거.
- **앨범별 센터**: 컴백 기획에서 이번 앨범의 센터를 선택(무드 선택 후 멤버별 어울림 등급 표시 — 환상의 조합/잘 어울림/무난/어색함). 앨범 완성도의 멤버 적합이 팀 평균 60% + 센터 40%로 바뀌고, 활동기 개인 인기 노출도 앨범 센터가 센터급으로 받는다. 청량 트렌드 주간에 순수×강아지상 멤버를 센터로 세우는 변주가 실제 수치로 이어진다.
