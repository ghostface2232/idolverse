## 0. 총평

**창단 플로우(프롤로그→투자사→스태프→시설→오디션→포지션)는 완성도가 높지만, 게임의 본체인 "데뷔 이후"로 가는 문이 UI에 하나도 없다.** 시스템 코드(앨범/차트/프로모션/시상식/투자사)는 상당 부분 작성되어 있으나 배선이 끊겨 있고, 배선된 부분도 **효과 키 계약 불일치**로 핵심 의사결정의 절반이 조용히 무효화된다. 현재 빌드는 정확히 말하면:

> "훌륭한 창단 시뮬레이터 + 결과가 보이지 않는 무한 트레이닝 스킵 버튼"

- 승리 조건 없음, 패배 조건 없음(자금 -∞ 허용), 데뷔 불가, 앨범 발매 불가, 수입 파이프라인 전체 미개통.
- 설계 철학("모든 선택에 기회비용")이 데이터↔엔진 키 불일치 하나로 붕괴된 상태.
- 반대로 좋은 소식: **시스템 레이어가 순수 함수 + 시드 주입으로 잘 설계되어 있어, 대부분의 문제가 "배선"과 "계약 강제"로 해결 가능**하다. 새로 짤 것보다 이을 것이 많다.

## 2. 치명 결함 (P0 — 교차 검증 완료)

### 2-1. 효과 키 계약 붕괴 ★최우선★
효과가 전부 `Record<string, number>`(types/game.ts:200,344,387)이고 적용기가 **5곳에 복제**되어 있으며(weekProcessor 4곳 + weekRunner.applyEffects), 각각 다른 키 집합을 처리한다.

- 결정 카드의 `trainee_vocal`, `trainee_dance`, `trainee_stress`, `album_song`, `album_marketing`, `fandom_loyalty`, `fandom_disappointment`, `trainee_specialist` (decisionCards.ts:15,22,44,51,58,87,103,110,117 / generateWeeklyDecisionCards.ts:115,169) → weekProcessor.ts:212-263은 camelCase 10종만 처리 → **전부 무음 폐기**.
- 결과: "보컬 집중 훈련"이 보컬을 안 올리고, "데모 곡 투자 -2,500만"·"개인 레슨 -600만"은 돈만 태우는 함정 카드. "파격 변신"은 리스크 키가 드롭되어 **무위험 고보상**(설계 의도와 정반대).
- 이벤트 자동효과(weekProcessor.ts:437-453)도 팬덤 5축만 처리 → "훈련 중 부상"(condition -18/stress +8), "멤버 갈등 폭발"(chemistry -15/satisfaction -8)이 이빨 빠진 상태.
- 선택형 이벤트는 base effects + choice effects **이중 적용** 의혹(weekProcessor 롤 시점 + weekRunner 해결 시점).

**수정안**: `EffectKey` union 타입 + `Partial<Record<EffectKey, number>>` 강제 + 단일 `applyEffects()` 유틸로 5곳 통합. 이 한 가지로 나머지 불일치를 tsc가 전부 찾아준다.

### 2-3. 경제 단위/수치 버그
- **월비→주차감**: founding.ts의 `monthlyCost`(월)를 FacilityInvestment.tsx:37-42가 `fixedCosts`로 넣고 economySystem.ts:69-70이 **매주** 차감. 스태프 월급도 동일. 실효 고정비 **4.33배**(최소 구성 ~1,480만/주).
- **부상 공식 오류**(trainingSystem.ts:122-127 + balance.ts:40-42): `0.02 + (100-stamina)×(-0.03) + stress×0.05` → 스트레스 40에서 52%, 50에서 100% 확정 부상. 게다가 **stamina가 높을수록 부상 확률이 오르는** 부호 반전. 계수 스케일 ~100배 오류.
- **potential 단위 불일치**: recruitSystem.ts:143은 0.8~1.7 배율로 생성, trainingSystem.ts:183은 `0.5 + potential/100`으로 소비 → 전원 성장 배율 ~0.51 고정. 유망주 스카우트 예산(최대 1억)의 가치 붕괴 + 전체 성장 속도 의도 대비 절반(스탯 40→70에 ~60주; 데뷔 목표 ~20주와 괴리).
- **어워드 100배 스케일 버그**(awardsSystem.ts:77-79): 경쟁자 지표가 `(fandom/100)*60`(→수천), 플레이어는 0~100 스케일 → 전 부문 라이벌 독식, 수상 구조적 불가. `*60`→`*0.6` 오타로 추정.

### 2-4. 투자사 시스템 = 데스 스파이럴 + 유일한 익스플로잇
- `musicShowWins: 0`, `adContracts: 0` 하드코딩(weekProcessor.ts:815,820) → 엔터·화장품 투자사 조건 **구조적 영구 실패**.
- 조건 하나라도 실패하면 해당 투자사의 **모든 페널티를 매주 전체 재적용**(weekProcessor.ts:579-625). VC `cashRecall`은 매주 -1.5억 → 13주차부터 회생 불가 강제 파산.
- `investorPenaltyActive`는 한 번 true면 **영구 고착**(리셋 코드 없음) → 매주 위기 카드 강제 등장 → "요구 수용" 선택 시 **+2,000만/주 무한 수입**(generateWeeklyDecisionCards.ts:86). 스태프 0명+시설 Lv1이면 순익 +1,450만/주. **"투자사 조건을 일부러 실패하는 것"이 유일한 흑자 전략.**
- 연도 랩 버그: `currentWeek < deadlineWeeks`(economySystem.ts:131)인데 주차가 52에서 1로 랩(advanceWeek.ts:5-7) → 마감 104(VC 회수)는 영원히 미평가. 수상 기록도 미영속이라 52주차 `awardLevel` 체크는 항상 null.
- 파산 처리 없음: 자금 음수는 경고 문자열 1줄(economySystem.ts:96-99). 게임오버 코드 0건.

- TypeScript strict 통과, `as any` 0건, non-null assertion 3건뿐. balance.ts의 상수마다 설계 의도 주석은 모범적.
- 창단 플로우 UX(투자사 스냅 캐러셀, 예산 밴드 미리보기, 포지션 적합도 %), 프롤로그 타이핑 연출, 버튼 눌림 물리, TraineeDetail 레이더 차트+대사.
- 모바일 기초 체력: h-dvh+safe-area, iOS 줌 방지, keep-all 줄바꿈, 바텀시트 패턴.
- Supabase 스키마(RLS, 슬롯 CHECK, 트리거) 건전.
- 이벤트 트리거 조건 체계(minWeek/minFame/보안팀 감면 등)와 선택지 트레이드오프 설계 자체는 충실.
