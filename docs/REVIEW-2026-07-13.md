## 1. 총평

**창단 플로우(프롤로그→투자사→스태프→시설→오디션→포지션)는 출시급 완성도지만, 게임의 본체인 "데뷔 이후"로 가는 문이 UI에 하나도 없다.** 시스템 코드(앨범/차트/프로모션/시상식/투자사)는 상당 부분 작성되어 있으나 배선이 끊겨 있고, 배선된 부분도 **효과 키 계약 불일치**로 핵심 의사결정의 절반이 조용히 무효화된다. 현재 빌드를 정확히 요약하면:

> **"훌륭한 창단 시뮬레이터 + 결과가 보이지 않는 무한 트레이닝 스킵 버튼"**

- 승리 조건 없음, 패배 조건 없음(자금 -∞ 허용), 데뷔 불가, 앨범 발매 불가, 수입 파이프라인 전체 미개통.
- 설계 철학("모든 선택에 기회비용")이 데이터↔엔진 키 불일치 하나로 붕괴된 상태.
- 반대로 좋은 소식: **시스템 레이어가 순수 함수 + 시드 주입 RNG로 잘 설계되어 있어, 대부분의 문제가 "배선"과 "계약(타입) 강제"로 해결 가능**하다. 새로 짤 것보다 이을 것이 많다.

## 3. 치명 결함 (P0)

여러 리뷰 트랙에서 독립적으로 교차 확인된 항목들이다.

### 3-1. 효과 키 계약 붕괴 ★최우선★

효과가 전부 `Record<string, number>`(`src/types/game.ts:200,344,387`)이고, 효과 적용기가 **5곳에 복제**되어 각각 다른 키 집합을 처리한다(weekProcessor 내 4곳: 결정 카드 212-263행 / 프로모션 299-321행 / 랜덤 이벤트 437-453행 / 투자사 페널티 584-624행, 그리고 `weekRunner.ts:98-174`의 `applyEffects`).

- 결정 카드 데이터의 `trainee_vocal`, `trainee_dance`, `trainee_stress`, `album_song`, `album_marketing`, `fandom_loyalty`, `fandom_disappointment`, `trainee_specialist` 키(`decisionCards.ts:15,22,44,51,58,87,103,110,117` / `generateWeeklyDecisionCards.ts:115,169`)는 weekProcessor가 camelCase 10종만 처리해 **전부 무음 폐기**된다.
- 결과: **"보컬 집중 훈련"이 보컬을 안 올리고**, "데모 곡 투자 -2,500만"·"개인 레슨 -600만"은 돈만 태우는 함정 카드가 되며, "파격 변신"은 리스크 키(`fandom_disappointment`)가 드롭되어 **무위험 고보상**(설계 의도와 정반대)이 된다.
- 이벤트 자동효과(`weekProcessor.ts:437-453`)도 팬덤 5축 키만 적용 → "훈련 중 부상"(condition -18/stress +8), "멤버 갈등 폭발"(chemistry -15/satisfaction -8)이 이빨 빠진 상태. 부정 이벤트 9종 중 4종이 실질 무해.
- 같은 키가 이벤트 선택지로 오면 적용되고(weekRunner 경유) 카드로 오면 무시되는 비결정적 계약. 선택형 이벤트는 base effects(롤 시점) + choice effects(해결 시점) **이중 적용** 의혹도 있다.

**수정안**: `type EffectKey = "money" | "public" | ...` union + `Partial<Record<EffectKey, number>>`로 전 데이터 파일 강제 + 단일 `applyEffects()` 유틸로 5곳 통합. 이 한 가지 조치로 나머지 불일치를 tsc가 컴파일 타임에 전부 찾아준다.

### 3-4. 투자사 시스템 = 데스 스파이럴 + 유일한 익스플로잇

- `musicShowWins: 0`, `adContracts: 0` **하드코딩**(`weekProcessor.ts:815,820`) → 엔터·화장품 투자사의 핵심 조건은 구조적 영구 실패.
- 조건 하나라도 실패하면 해당 투자사의 **모든 `penaltyEffects`를 매주 전체 재적용**(`weekProcessor.ts:579-625`). VC의 `cashRecall`은 매주 -1.5억(투자금의 10%, `economySystem.ts:213`) → **13주차부터 회생 불가 강제 파산**.
- `investorPenaltyActive`는 한 번 true면 **영구 고착**(리셋 코드 없음, `weekProcessor.ts:566-582`) → 매주 위기 카드 강제 등장 → "요구 수용" 선택 시 **+2,000만/주 무한 수입**(`generateWeeklyDecisionCards.ts:86`). 스태프 0명 + 시설 Lv1이면 순익 +1,450만/주 영구. **"투자사 조건을 일부러 실패하는 것"이 유일한 흑자 전략**인 도착적 인센티브 구조.

### 3-6. 세이브 안정성

- **industry 자기강화 루프**: `musicQualityHigh = industry > 50`(`weekProcessor.ts:547`) → `fandomSystem.ts:85`에서 industry +3/주 → **50만 넘으면 아무것도 안 해도 100까지 직진**, 50 이하에선 부트스트랩 불가.
- **currentActivity sticky(주간 리셋 없음)**: 멤버 1명을 예능에 영구 배치하면 매주 public +4 + 비활동 감쇠 면제(`weekProcessor.ts:531-534` isActive) — 지배 전략. rest 영구 배치 = 만족도 +5/주 무한.
- **케미 단조 증가**: 훈련 그룹 전 쌍 +2/주, 갈등 쌍(-50 미만)도 기대값 +0.9/주(30% 확률 +10, 70% -3) → **감소 요인이 사실상 없어** 수십 주 내 +100 포화, 긴장 소멸.

3. "요구 수용" = +2,000만/주, 상한 없음 →
4. 스태프 0명(고용 강제 없음, 경고 모달뿐) + 시설 Lv1이면 **순이익 +1,450만/주 영구**. 부작용(실망도 +5/주)은 팬덤 0이라 실질 무의미.
